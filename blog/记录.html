# Vue SSR 客户端激活 - 事件绑定

在初始化 patch 函数时，会收集所有的事件 hook 到 cbs 中，如下格式：

cbs = {
  activate: [ƒ]
  create: (8) [
    updateAttrs(oldVnode, vnode),
    updateClass(oldVnode, vnode),
    updateDOMListeners(oldVnode, vnode),
    updateDOMProps(oldVnode, vnode),
    updateStyle(oldVnode, vnode),
    _enter(_, vnode),
    create(_, vnode),
    updateDirectives(oldVnode, vnode)
  ],
  destroy: (2) [ƒ, ƒ]
  remove: [ƒ]
  update: (7) [ƒ, ƒ, ƒ, ƒ, ƒ, ƒ, ƒ]
}

对 vue 组件模版进行编译时，会将 DOM 节点上的属性（attr、事件等）放到 data 中，如下格式：

vnode = {
  data: {
    on: {click: ƒ}
  }
}

在客户端激活时，会调用 hydrate 函数对客户端 vnode 和服务端 vnode 进行匹配混合，之后会对 data 进行遍历，接着调用 invokeCreateHooks，通过 cbs.create 中的所有函数执行 data-item 逻辑。

在客户端事件绑定激活过程。在对 vnode data 中数据进行遍历后，判断当前属性是否未在服务端已渲染，事件在服务端没有绑定 DOM 节点，所以会执行并调用 invokeCreateHooks，在 invokeCreateHooks 函数内部通过遍历并执行 cbs.create 下的所有函数包含 updateDOMListeners，实现事件绑定到 DOM 节点。

源码如下：

function hydrate(elm, vnode, insertedVnodeQueue, inVPre) {
  // ...
  {
    if (!assertNodeMatch(elm, vnode, inVPre)) {
      return false
    }
  }
  // ...
  // 绑定渲染服务端未执行的 attr
  if (isDef(data)) {
    var fullInvoke = false
    for (var key in data) {
      if (!isRenderedModule(key)) { // 判断是否在服务端已执行
        fullInvoke = true
        invokeCreateHooks(vnode, insertedVnodeQueue)
        break
      }
    }
    // ...
  }
  // ...
}

function invokeCreateHooks(vnode, insertedVnodeQueue) {
  for (var i$1 = 0; i$1 < cbs.create.length; ++i$1) {
    cbs.create[i$1](emptyNode, vnode) // 调用 cbs.create 下的所有函数，如：updateDOMListeners
  }
  i = vnode.data.hook // Reuse variable
  if (isDef(i)) {
    if (isDef(i.create)) {
      i.create(emptyNode, vnode)
    }
    if (isDef(i.insert)) {
      insertedVnodeQueue.push(vnode)
    }
  }
}

# Nodejs Event Loop

  • Timer: setTimeout setInterval（在 poll 空闲时执行）
  • Pending Callback: 上一轮遗留的 callBack，执行一些错误的系统回调。
  • Idle、prepare: nodejs 内部调用
  • poll: I/O等
    • 轮询队列不为空
      • 执行轮询队列，之后执行 check
    • 轮询队列为空
      • 如果有 setImmediate 在等待执行，则会执行 check
      • 没有 setImmediate，如果有超时的 timer，则会执行超时的 timer，没有则阻塞挂起该阶段，下一轮 loop 会去 timer 检查有无需要执行的 timer (opens new window)
  • check: setImmediate（在 poll 执行完之后执行）
  • close callback

在每次事件循环运行之间，Node.js会检查它是否正在等待任何异步I/O或timers，如果没有，则将其干净地关闭。

process.nextTick 独立的队列，会先清空该队列再执行 event loop。

通过 v8 引擎解析 js 代码，然后解析后的代码调用 node API，node API 通过 libuv 执行，最后通过 v8 引擎返回执行结果。
