# Vue 组件粒度精确更新

当 vue 对组件实例进行更新时，对于不是组件的元素节点，会通过 patchVnode 方式进行更新；对于子组件节点，则会调用 prepatch - updateChildComponent 方法对子组件的 props、listenters 等进行更新，不会直接更新子组件。

因为在父组件渲染子组件的时候，会将 props 传递给子组件，保存在子组件的 _props 中，在子组件初始化时会对 props 做响应式处理，子组件对 props 的获取触发了 getter，这样 props 就收集到了子组件的 render-watcher。

当父组件 _update 的时候，重新计算 props，触发了 props 的 setter，进而通知自组件 render-watcher 更新。

$forceUpdate 实际上只是触发了 render-watcher 的重新执行，和修改一个响应式属性触发更新的原理是一样的，都是调用 _watcher.update()。【门面模式】

# Vue 生命周期

# beforeCreate / created：实现 Vue 实例的初始化。

beforeCreate：初始化组件的生命周期、事件机制和 render 函数。

created：初始化 props、data、method 等，完成数据响应式处理。

initLifecycle(vm);
initEvents(vm);
initRender(vm);
callHook(vm, 'beforeCreate');
initInjections(vm); // resolve injections before data/props
initState(vm);
initProvide(vm); // resolve provide after data/props
callHook(vm, 'created');

# beforeMount / mounted：实现 Vue 实例的挂载

beforeMount：在 $mount->mountComponent一开始就调用执行了，这时还没实例化渲染 Watcher，也就是还没调用 render 函数创建 vnode 和调用 update 执行 patch。

mounted：已实例化渲染 Watcher，也就是调用了 render 函数创建 vnode 和调用 update 执行 patch 并挂载到生成实际的DOM 元素了。

# beforeUpdate / update：响应式数据更新时触发

响应式数据更新时，通知对应的 Watcher 执行对应的更新操作。（通过异步队列缓存进行异步更新 DOM）
